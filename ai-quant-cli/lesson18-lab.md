# 第 18 节 实验手册：从空目录造一个 AI 量化投研系统

> 配套课程：AI 业务流架构师 · 第 18 节《AI Quant CLI 量化投研系统开发》
> 预计耗时：60–90 分钟
> 操作方式：全程在终端和 Claude Code 对话完成——**你给目标、做设计 review，它动手**；遇报错让它自愈，你不用自己背命令
> 前置条件：Claude Code 已装好并接 Opus 4.8（第 16 节内容）；本机有 Python 3.10+；准备一份 A 股年报 PDF

---

## 0. 开始前确认

| # | 物料 | 备注 |
|---|---|---|
| 1 | Claude Code 能正常对话 | `claude` 进入后能对话；驱动模型用 Opus 4.8（订阅 `/login`，或中转站配 env） |
| 2 | Python 3.10+ | `python3 --version` 能正常输出 |
| 3 | 一份 A 股年报 PDF | 从巨潮资讯网按股票代码搜「年度报告」下载（下载说明见本仓库 `data/README.md`） |
| 4 | 一个你自己的空目录 | 如 `~/my-quant/`——**在这里做，不要在本仓库的 `ai-quant-cli/` 里做，那是做好的参考答案** |

> 本节从**空目录**起步：不是改现成代码，是让 Claude Code 自己设计、施工、自愈，造出一套能跑的系统。做完拿本仓库 `ai-quant-cli/` 的 `src/` + `CLAUDE.md` 当参考答案对照。

---

## 1. 起步：建目录、放年报、启动 Claude Code

```bash
mkdir ~/my-quant && cd ~/my-quant
mkdir data
# 把下载好的年报 PDF 放进 data/
claude
```

启动后登录（或配好中转站 env），切到 **Auto 模式**——本节让 Claude Code 自己施工、自愈，Auto 模式不必每步手动确认。

> 💡 省事版：把本仓库的 `BUILD_PLAN.md` 拷进 `~/my-quant/`，直接发「读 BUILD_PLAN.md，一步步把系统做出来，每阶段停一下让我看」，它会自己跑完六个阶段。想一步步体会方法论，就照下面来。

---

## 2. 先设计，别写代码（你是架构 reviewer）

从空目录造系统，第一步不是写代码，是让 Claude Code 出设计、你来 review。发送目标：

```text
我要从零做一个本地量化投研系统：把 data/ 里的年报 PDF，自动解析三张合并报表、
由你做财务风险研判、出图、生成一份带图表的 HTML 投研报告。
先别写代码——先给我系统设计：分层架构、各层模块与职责、层间数据契约、
技术选型、目录结构、你预见的风险点。
约定：分析判断由你亲自做，Python 脚本只做确定性工作（解析 / 出图 / 汇编 HTML），
不要在代码里调用大模型 API。
```

读一遍方案，提一句你想改的（例：「中间结构化数据落盘成 JSON 文件，方便每层单独调试重跑」），确认后让它搭项目骨架，再自己敲 `/init` 生成 `CLAUDE.md`。

> 💡 学习点：从零造系统，**先设计后施工**——先让它出蓝图、你拍板，再动手。`/init` 把设计沉淀进 `CLAUDE.md`，下次对话一上来就懂，不用重讲。
> ⚠️ 关键约定要在这一步就说死：**分析判断它亲自做、代码不调大模型 API**。这决定了整套架构——智能在它的循环里，不在某个 API 调用里。

---

## 3. 解析财报：定位三张合并报表（自愈现场）

```text
写解析脚本：从 data/ 的年报 PDF 定位三张合并报表（资产负债表 / 利润表 / 现金流量表），
把关键科目连同期末、期初抽成结构化数据落盘到 data/parsed/；
抽完用会计恒等式（资产 = 负债 + 所有者权益）自检。
先只解析、不分析，遇报错你自己修到跑通。
```

这一步是**自愈主场**。年报有两百多页、还同时含「母公司」和「合并」两套三表，Claude Code 通常会接连撞上几个坑再自己修好：

- **缺包**：装 `pdfplumber` 等依赖
- **抓错表**：第一版可能抓到「母公司」报表或附注里的同名表——让它锁定**合并**报表
- **中文数字**：千分位逗号、单位（千元 / 万元）、空行——清洗成干净数值

> 💡 学习点：**给目标不给步骤**。你没教它怎么解析 PDF，它在报错里自己查、自己修。会计恒等式自检（资产 = 负债 + 所有者权益）是它判断「抽对没有」的硬标尺——恒等式不平，就说明抽漏或抽错了。
> ⚠️ 看到它撞坑别急着接管，让它自愈——这正是本节要练的肌肉。

---

## 4. 风险研判 + 三表勾稽（Claude Code 亲自当分析师）

解析出干净数据后，让它做**研判**——这一步不写脚本调 API，是 Claude Code 自己读数、自己下判断：

```text
基于 data/parsed/ 的结构化数据做财务风险研判（应收 / 存货 / 盈利质量 /
资本开支 / 集中度等），每条结论给出数据依据；
再做三表勾稽交叉验证（如净利润 vs 经营现金流、营收增速 vs 应收账款增速），
每条给依据。把结论存成 analysis/findings 文件。
```

挑一条结论追问它，验证「可解释」：

```text
这条结论你是从哪几张表、哪些科目推出来的？算给我看看。
```

> 💡 学习点：这是本节和「写脚本调大模型 API」最大的区别——**智能在 Claude Code 的 agent 循环里**。它能边读数边推理、还能被你追问每条结论的依据（指向具体报表 + 科目 + 期间）。三表勾稽能挖出单看一张表发现不了的风险（如净利润涨、经营现金流却没跟上）。

---

## 5. 出图：中文乱码的经典自愈

```text
用 data/parsed/ 的结构化数据，给关键指标（营收、净利润、毛利率、现金流、
存货、应收等）出趋势图 / 结构图，保存成图片供 HTML 报告嵌入，遇报错你自己修。
```

修好后追一句，把经验沉淀下来：

```text
把这次中文字体的修法写回 CLAUDE.md，以后出图默认遵守。
```

> 💡 学习点：matplotlib 默认不带中文字体，第一版图里中文几乎一定是**方框 □□□**、负号也可能乱码。Claude Code 会自己发现、挑一个本机可用的中文字体修好——你别急着帮它。修好后让它把字体配置写回 `CLAUDE.md`，下次出图不再踩同一个坑，这就是「自愈 + 记忆」。

---

## 6. 生成 HTML 报告

```text
把结构化数据、图表、风险研判、跨表结论汇编成一个 HTML 投研报告网页：
排版清晰、图表内嵌、浏览器可直接打开，输出到 reports/。
```

浏览器打开 `reports/` 下的网页看成品——这才是交付物：是网页不是聊天框里的一段文字，有图表、有研判、可复用。

---

## 7. 一键端到端：把整条线串起来

```text
把整条流程串成一个一键入口：给一句指令 + 指定 data/ 里的年报，
就自动跑完 解析 → 出图 → 研判 → 生成 HTML 报告。用现有年报端到端跑一遍。
```

> 💡 学习点：到这里你交付的不是「一份报告」，是「一套能反复跑的系统」。「重跑 = 再喊一次它」——换标的、换一期，一句话再来一份。注意：研判（findings）由它亲自产出，一键编排里它会先确认研判到位再出报告，缺了不会硬出半成品。

---

## 课后作业

1. **必做**：换一只你关注的标的（或同公司换一期年报），用你造的系统重跑一份 HTML 投研报告，体会「造好一次、反复复用」。
2. **进阶**：给系统加一类新分析（如同业对比 / 多期趋势），走一遍「改设计 → 改 CLAUDE.md → 施工」的闭环——巩固「先设计后施工」。
3. **选做**：对照本仓库 `ai-quant-cli/` 的 `src/` + `CLAUDE.md`，看你和参考答案差在哪，尤其是分层与数据契约。

## 小结

- **先设计后施工**：空目录起步，先让它出架构蓝图、你 review，再逐层施工，设计固化进 `CLAUDE.md`。
- **智能在循环里**：财务研判它亲自做、可追问可解释，不是写脚本调大模型 API。
- **自愈**：给目标不给步骤，缺包 / 抓错表 / 中文乱码，让它在报错里自己修——修法写回 `CLAUDE.md`，越用越顺。
- **交付的是系统不是报告**：换标的一句话重跑，这套可复用系统正是下一节（L19）夜间自动化的输入。

从「改已有代码」到「从空目录造一套能跑的系统」——这一节你迈过去了。
