# AI Quant CLI · 自动构建计划（交给 Claude Code 执行）

> **用途**：把这份计划交给 Claude Code，让它自己一步步把整个系统做出来。你只在两个标 **【需你输入】** 的点介入，其余时间在每个阶段的暂停点截图即可。
>
> **怎么用**：
> 1. 本文件已在 `ai-quant-cli/` 就位。把宁德时代年报 PDF 放进同目录的 `data/`（见 `data/README.md`）。
> 2. 在 `ai-quant-cli/` 启动 `claude`，登录好（订阅或中转站），选到 **Auto 模式**（mode 4：它自己跑、改文件和跑命令都不每次问你，正是这套自动构建要的）。
> 3. 发这一句启动：
>    > 读 BUILD_PLAN.md，按里面的阶段一步步把这个系统做出来。每完成一个阶段就停下、告诉我「阶段 N 完成，可截图」，等我说「继续」再做下一阶段。遇到标【需你输入】的地方停下问我。
> 4. 之后你基本只用：看一眼 → 截图 → 说「继续」。两个 **【需你输入】** 点按提示给一句话。

## 总目标
从 `data/` 里的年报 PDF，造一个本地系统：解析三张合并报表 → **你（Claude Code）亲自** 做财务风险研判 → 出图 → 生成 HTML 投研报告。
**铁律**：财务分析判断由你直接做；Python 脚本只做确定性工作（解析 PDF、出图、汇编 HTML），**不要在代码里调用大模型 API**。遇到报错你自己调试到跑通，调试过程留在终端里、别清屏。

## 阶段 1 · 系统设计　【需你输入】
- 先别写代码。给出系统设计：分层架构、各层模块与职责、层间数据契约、技术选型（讲理由）、目录结构、你预见的风险点。
- 给完 → 停，等我 review（我可能提一条修正）。
- 我确认后：按设计搭项目骨架（目录 + 各模块空文件 + `.gitignore` 排除 `data/` 下的 PDF / 数据），搭完停一下——**`/init` 由我来敲**（slash 命令只有我能发）；你据此把 CLAUDE.md 补全，写入设计与约定（含「分析由你做、不调 API」「报告输出到 reports/」）。
- 停：「阶段 1 完成，可截图」。

## 阶段 2 · 财报解析
- 写解析脚本：从年报 PDF 里定位三张合并报表（资产负债表 / 利润表 / 现金流量表），把关键科目连同期末、期初数抽成结构化数据，落盘到 `data/`。抽完用会计恒等式（资产 = 负债 + 所有者权益）自检对不对。先只做解析，不分析。
- 遇报错自己修到跑通（缺包、表格定位、数字格式等都自己处理）。
- 停：「阶段 2 完成，可截图」。

## 阶段 3 · 风险研判　【可能追问】
- 基于结构化数据，做隐性财务风险研判（应收 / 存货 / 盈利质量 / 资本开支 / 集中度等），每条给数据依据。
- 再做三表勾稽交叉验证（如净利润 vs 经营现金流、营收增速 vs 应收账款增速），每条给依据。
- 我可能现场追问某条结论从哪几张表、哪些科目推的——你解释清楚。
- 停：「阶段 3 完成，可截图」。

## 阶段 4 · 可视化出图
- 写出图脚本，把关键指标（营收、净利润、毛利率、现金流、存货、应收等）做成趋势图 / 结构图，保存成图片供 HTML 报告嵌入。遇报错自己修。
- 把你在这一步解决的任何渲染配置写回 CLAUDE.md，以后这个项目出图默认遵守。
- 停：「阶段 4 完成，可截图」。

## 阶段 5 · 生成 HTML 报告
- 把结构化数据、图表、风险研判、跨表结论汇编成一个 HTML 投研报告网页：排版清晰、配色专业、图表内嵌、浏览器可直接打开，输出到 `reports/`。
- 停：「阶段 5 完成，可截图」。

## 阶段 6 · 端到端编排
- 把整条流程串成一个可一键重跑的入口：给一句指令 + 指定 `data/` 里的年报，就自动跑完 解析 → 研判 → 出图 → 生成 HTML 报告。用现有年报端到端跑一遍。
- 停：「阶段 6 完成，可截图。全部完成」。

### 写这份计划时刻意「只给目标、不给做法」（重要）
本计划不告诉 Claude Code 用哪个库、怎么定位表、数字什么格式、怎么配中文字体——就是要它自己撞、自己修，这些自愈过程才是课程要截的图。所以**别往这份文件里加任何「解决方案」**（否则它照着做、不踩坑，就没自愈可演了）。
